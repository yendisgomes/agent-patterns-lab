Agent Patterns Lab

Esta demo reencena, com transcrições reais já validadas contra a API do Groq, os 6 padrões de agente do artigo Building Effective Agents, todos aplicados ao mesmo domínio — rastreamento e faturamento de pedidos — para permitir comparar lado a lado como cada padrão resolveria o mesmo problema. Nenhuma chamada de API acontece no navegador: os passos abaixo são exatamente o que os scripts em patterns/ produziram quando rodados de verdade.

01

Augmented LLM

LLM + tool-calling nativo, sem workflow. O ponto de partida recomendado pelo artigo.

Ver demo →
02

Prompt Chaining

Extração → gate programático → tools → síntese. Etapas sequenciais fixas.

Ver demo →
03

Routing

Classificador decide a categoria; código puro roteia para o prompt especializado.

Ver demo →
04

Parallelization

Sectioning (subtarefas fixas em paralelo) + Voting (consenso por maioria).

Ver demo →
05

Orchestrator-Workers

Decomposição dinâmica — o número de subtarefas é decidido em tempo de execução.

Ver demo →
06

Evaluator-Optimizer

Loop gerador↔avaliador — guardrail automatizado da política anti-alucinação.
